시스템 구성은.

액터 → A 요청 → 

- 관리 객체 case 1 → 바로 A 처리. (바로 처리 가능한 경우. [ex. 검색하기] )

- 관리 객체 case 2 → A.1 요청 → 

실행 객체 → A.1 실행.




예시). 로그인

회원은 로그인을 요청 한다.

로그인 객체는 요청을 받아 요청이 고정or익명 인지 구분하여 각각에 해당하는 메소드를 실행.

고정 로그인 요청의 경우 -> 고정 로그인 정보 객체에 접근하여 ID/PW 체크하고 True 키 반환.

익명 로그인 요청의 경우 -> 익명 로그인을 위한 IP 정보를 사용자의 컴퓨터에서 제공 받아 기록 후 False 키 반환.

- T/F 키는 사용자의 로그인 상태를 고정or익명으로 구분하는 키.
