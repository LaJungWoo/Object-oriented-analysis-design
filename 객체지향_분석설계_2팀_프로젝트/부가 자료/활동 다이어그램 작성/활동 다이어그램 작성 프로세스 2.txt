1. 작성 대상 : 고정 로그인 유즈케이스

2. 활동 상태 :

	2.1 사용자가 회원 UI를 이용해 로그인을 시도한다.
	2.2 회원 UI는 로그인 요청을 로그인 관리 객체로 전달한다.
	2.3 로그인 관리 객체는 고정로그인 실행 객체에 고정로그인을 요청한다.
	2.4 고정 로그인 실행 객체는 ID/PW를 입력받아 체크하고 승인 키를 관리 객체로 리턴.
	2.5 관리 객체가 승인 키를 받아 정지/벤 상태를 체크한 승인 키를 회원 UI로 리턴
	2.6 회원 UI가 로그인 성공 승인 키를 받으면 성공.


3. 모델링.

사전 조건 : 
	- 사용자 인증 단말 확인.

종료 조건 : 로그인 5회 이상 실패.

사후 조건 : 
	1. 60초간 로그인 접근 금지. 
	2. 홈페이지 메인으로 이동

분기 조건 : 
	1. 로그인 방법 자동/수동 선택.
		- 로그인 시 자동 로그인/ 수동 로그인 설정 가능.
	2. 로그인 실패/성공.
		- 로그인 실패시 : 실패 메시지 띄움.
		- 로그인 성공시 : 패스.
	3. ID 정지의 경우.
		- 해당 ID가 정지 당한 경우 정지사유 팝업.
		- 미정지시 패스.

동기화 조건 : ID입력, PW입력. ID체크, PW체크.

Swimlanes :
	1. 사용자. 
	2. 관리객체
	3. 실행객체
	4. 외부 시스템.
